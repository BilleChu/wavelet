# 专业回测评估平台文档

## 概述

本系统是一个专业级的量化策略回测评估平台，提供完整的策略评估流程和深度分析报告。

## 系统架构

### 1. 核心组件

#### 1.1 回测引擎 ([engine.py](file:///Users/binzhu/Projects/wavelet/backend/openfinance/quant/backtest/engine.py))
- **功能**: 执行策略回测的核心引擎
- **特点**:
  - 支持多种调仓频率（日度、周度、月度、季度）
  - 真实的交易成本模拟（佣金、滑点）
  - 完整的持仓和交易记录
  - 基准对比分析

#### 1.2 绩效指标计算器 ([metrics.py](file:///Users/binzhu/Projects/wavelet/backend/openfinance/quant/backtest/metrics.py))
- **功能**: 计算60+专业绩效指标
- **指标类别**:
  - **收益指标**: 总收益、年化收益、超额收益、CAGR
  - **风险指标**: 波动率、下行波动率、VaR、CVaR、最大回撤
  - **风险调整收益**: 夏普比率、索提诺比率、卡尔马比率、信息比率
  - **市场风险**: Alpha、Beta、R²、跟踪误差
  - **交易统计**: 胜率、盈亏比、换手率、平均持仓周期
  - **高级指标**: 偏度、峰度、尾部比率、Ulcer指数

#### 1.3 报告生成器 ([report_generator.py](file:///Users/binzhu/Projects/wavelet/backend/openfinance/quant/backtest/report_generator.py))
- **功能**: 生成专业级回测报告
- **报告内容**:
  - 执行摘要（关键指标、策略评级）
  - 详细绩效指标
  - 风险分析
  - 收益分析
  - 回撤分析
  - 交易分析
  - 基准对比
  - 月度/年度收益分解
  - 风险归因
  - 优化建议

#### 1.4 完整工作流 ([complete_workflow.py](file:///Users/binzhu/Projects/wavelet/backend/openfinance/quant/backtest/complete_workflow.py))
- **功能**: 端到端的回测流程
- **流程步骤**:
  1. 初始化组件
  2. 加载策略配置
  3. 准备历史数据
  4. 计算因子值
  5. 执行回测
  6. 生成报告
  7. 保存结果

## 使用方法

### 1. 运行完整回测

```python
from openfinance.quant.backtest.complete_workflow import StrongStockBacktestWorkflow
from datetime import date, timedelta

# 初始化工作流
workflow = StrongStockBacktestWorkflow()
await workflow.initialize()

# 加载策略
await workflow.load_strategy("strategy_strong_stock")

# 运行回测
results = await workflow.run_backtest(
    start_date=date.today() - timedelta(days=365),
    end_date=date.today(),
    initial_capital=1000000.0,
)

# 打印报告
workflow.print_executive_summary(results["report"])
workflow.print_performance_metrics(results["report"])
workflow.print_risk_analysis(results["report"])

# 保存报告
workflow.save_report(results["report"])
```

### 2. 命令行运行

```bash
cd /Users/binzhu/Projects/wavelet/backend

```

## 专业指标说明

### 1. 收益指标

#### 总收益率 (Total Return)
- **定义**: 整个回测期间的总收益
- **公式**: (最终权益 - 初始资金) / 初始资金
- **意义**: 衡量策略的绝对收益能力

#### 年化收益率 (Annual Return)
- **定义**: 年化后的平均收益率
- **公式**: (1 + 总收益率)^(252/天数) - 1
- **意义**: 便于与其他投资标的比较

#### 超额收益 (Excess Return)
- **定义**: 超过基准的收益
- **公式**: 策略收益 - 基准收益
- **意义**: 衡量策略的主动管理能力

### 2. 风险指标

#### 年化波动率 (Volatility)
- **定义**: 收益率的标准差（年化）
- **公式**: std(日收益率) × √252
- **意义**: 衡量收益的波动程度

#### 下行波动率 (Downside Volatility)
- **定义**: 负收益的标准差（年化）
- **公式**: std(日收益率 < 0) × √252
- **意义**: 只关注下行风险

#### 最大回撤 (Max Drawdown)
- **定义**: 从峰值到谷值的最大跌幅
- **公式**: max((峰值 - 当前值) / 峰值)
- **意义**: 衡量最坏情况下的损失

#### VaR (Value at Risk)
- **定义**: 在给定置信水平下的最大可能损失
- **公式**: percentile(收益率, 5%)
- **意义**: 风险管理的重要指标

#### CVaR (Conditional VaR)
- **定义**: 超过VaR的平均损失
- **公式**: mean(收益率 < VaR)
- **意义**: 衡量尾部风险

### 3. 风险调整收益

#### 夏普比率 (Sharpe Ratio)
- **定义**: 单位风险的超额收益
- **公式**: (年化收益 - 无风险利率) / 年化波动率
- **评级**:
  - > 2.0: 优秀
  - 1.5-2.0: 良好
  - 1.0-1.5: 一般
  - 0.5-1.0: 较差
  - < 0.5: 很差

#### 索提诺比率 (Sortino Ratio)
- **定义**: 单位下行风险的超额收益
- **公式**: (年化收益 - 无风险利率) / 下行波动率
- **意义**: 只考虑下行风险的风险调整收益

#### 卡尔马比率 (Calmar Ratio)
- **定义**: 年化收益与最大回撤的比值
- **公式**: 年化收益 / 最大回撤
- **意义**: 衡量回撤风险下的收益能力

#### 信息比率 (Information Ratio)
- **定义**: 主动收益与跟踪误差的比值
- **公式**: (策略收益 - 基准收益) / 跟踪误差
- **意义**: 衡量主动管理能力

### 4. 市场风险指标

#### Beta
- **定义**: 策略对市场波动的敏感度
- **公式**: cov(策略收益, 市场收益) / var(市场收益)
- **意义**:
  - Beta > 1: 策略波动大于市场
  - Beta < 1: 策略波动小于市场

#### Alpha
- **定义**: 超过市场风险调整后的收益
- **公式**: 策略收益 - [无风险利率 + Beta × (市场收益 - 无风险利率)]
- **意义**: 衡量选股能力

#### R²
- **定义**: 策略收益与市场收益的相关性
- **公式**: corr(策略收益, 市场收益)²
- **意义**: 衡量策略与市场的关联度

### 5. 交易统计

#### 胜率 (Win Rate)
- **定义**: 盈利交易占总交易的比例
- **公式**: 盈利交易数 / 总交易数
- **意义**: 衡量交易准确性

#### 盈亏比 (Profit/Loss Ratio)
- **定义**: 平均盈利与平均亏损的比值
- **公式**: 平均盈利金额 / 平均亏损金额
- **意义**: 衡量单笔交易的收益质量

#### 换手率 (Turnover Rate)
- **定义**: 交易金额与平均资金的比值
- **公式**: 总交易金额 / (2 × 平均资金)
- **意义**: 衡量交易频率

## 最佳实践

### 1. 回测设置
- **回测周期**: 建议至少1年以上，最好3-5年
- **初始资金**: 根据实际资金规模设置
- **交易成本**: 使用真实的佣金和滑点数据
- **基准选择**: 选择合适的基准指数（如沪深300）

### 2. 数据质量
- **数据来源**: 使用高质量的历史数据
- **数据清洗**: 处理缺失值、异常值
- **复权处理**: 使用后复权价格
- **停牌处理**: 正确处理停牌股票

### 3. 风险控制
- **止损机制**: 设置合理的止损线
- **仓位管理**: 控制单只股票权重
- **行业集中度**: 避免过度集中于单一行业
- **流动性风险**: 考虑成交量限制

### 4. 性能优化
- **参数优化**: 避免过度拟合
- **样本外测试**: 保留部分数据做验证
- **稳健性检验**: 测试不同市场环境
- **敏感性分析**: 测试参数变化的影响
